How DNS Works in Active Directory

Domain Name System — the phonebook of your network

What is DNS?

DNS = Domain Name System. Computers talk to each other using IP addresses (like 192.168.1.10), but humans remember names (like SERVER01 or google.com). DNS is the translator between the two.

Think of it like your phone's contacts list. You tap "Mom" — your phone looks up her number and calls it. DNS does the same: you type SERVER01, your computer asks DNS, DNS replies "that's 192.168.1.10", and your computer connects.

Why DNS is Critical in Active Directory

In a Windows domain, DNS is not optional — it is the backbone of the entire AD structure. When you install Active Directory, DNS gets installed automatically alongside it. Here's why:

Domain Join
Computers find the Domain Controller via DNS
Login
When you log in, DNS locates the DC to authenticate
Group Policy
Clients use DNS to find sysvol/netlogon shares
Replication
DCs find each other via DNS to sync data

DNS Zones — What Are They?

A DNS Zone is like a folder that holds all the records for a domain. When you install AD with domain company.local, a zone called company.local is automatically created.

Zone TypeWhat It DoesExample
Forward Lookup ZoneConverts Name → IPfileserver → 192.168.1.20
Reverse Lookup ZoneConverts IP → Name192.168.1.20 → fileserver
💡 Beginner Tip: The DNS server in a Windows AD domain is almost always your Domain Controller itself. When you set up client PCs, their DNS setting should point to the DC's IP address — never to 8.8.8.8 (Google) as the primary DNS, or domain joining will fail.

Creating A Records Manually

Map hostnames to IP addresses

What Is an A Record?

An A record (Address record) is the most basic DNS record. It simply says: "This name lives at this IP address."

Example: PRINTER01 → 192.168.1.50

Without this record, other computers can't find PRINTER01 by name — they'd have to type the IP address directly every time.

When Do You Create A Records Manually?

Computers that join the domain usually register their own A records automatically. But you create them manually for devices that can't do this themselves:

Network Printers
Don't join the domain
NAS Devices
Linux-based, no AD agent
IP Cameras
Simple embedded devices
Web Servers
Might need custom names

Method 1 — Using DNS Manager (GUI)

1

Open DNS Manager

On your Domain Controller: Start → Windows Administrative Tools → DNS. Or press Win+R and type dnsmgmt.msc

2

Navigate to Your Zone

Expand your server name → Forward Lookup Zones → click your domain (e.g., company.local)

3

Create New Host (A) Record

Right-click on your zone → New Host (A or AAAA)...

4

Fill in the Details

Enter: Name: PRINTER01  |  IP Address: 192.168.1.50. Tick "Create associated PTR record" if you want reverse lookup too. Click Add Host.

Method 2 — Using PowerShell (Faster!)

PowerShell — Run as Administrator on DC
# Add a single A record
PS C:\> Add-DnsServerResourceRecordA -ZoneName "company.local" -Name "PRINTER01" -IPv4Address "192.168.1.50"

# Verify it was created
PS C:\> Resolve-DnsName "PRINTER01.company.local"

# List all A records in a zone
PS C:\> Get-DnsServerResourceRecord -ZoneName "company.local" -RRType "A"

Testing Your A Record

Command Prompt — From any PC on the network
# Test DNS resolution (basic)
C:\> ping PRINTER01

# Detailed DNS lookup
C:\> nslookup PRINTER01

# If it resolves, you'll see something like:
# Server: DC01.company.local
# Address: 192.168.1.1
# Name: PRINTER01.company.local
# Address: 192.168.1.50
⚠️ Common Mistake: If you create an A record but the device has a different IP than what you entered, DNS will point to the wrong place. Always double-check the IP before creating the record. For devices that might change IPs, use DHCP reservations (covered in the next section!).

Installing the DHCP Server Role

Automate IP address assignment across your network

What is DHCP?

DHCP = Dynamic Host Configuration Protocol. Without DHCP, you would have to manually set an IP address on every single computer, printer, and device on your network. With DHCP, devices connect to the network and automatically receive an IP address, subnet mask, gateway, and DNS server.

Think of DHCP as a hotel receptionist. You arrive (connect to network), they assign you a room (IP address), give you a key card (configuration), and tell you when checkout is (lease time). If you come back tomorrow, you might get the same room or a different one.

What DHCP Gives a Client Device

SettingExamplePurpose
IP Address192.168.1.101Device's unique identifier on the network
Subnet Mask255.255.255.0Defines the size of the network
Default Gateway192.168.1.1Router IP — how to reach the internet
DNS Server192.168.1.1 (DC)Where to resolve names
Lease Duration8 daysHow long the device keeps this IP

Install DHCP via Server Manager (GUI)

1

Open Server Manager

Should already be open. If not: Start → Server Manager (it's pinned to the taskbar).

2

Add Roles and Features

Click "Manage" (top right) → "Add Roles and Features"

3

Step through the wizard

Click Next → Next → Select your server → Check "DHCP Server" → click "Add Features" in the popup → Next → Next → Install

4

Wait for Installation

Takes 1-2 minutes. You'll see "Installation succeeded." Do NOT restart yet — the next step requires a post-install configuration.

5

Complete DHCP Configuration

A yellow flag appears in Server Manager (top right bell icon). Click it → "Complete DHCP configuration" → Next → Commit. This creates the DHCP Administrators and DHCP Users local security groups.

Install DHCP via PowerShell (Faster)

PowerShell — Run as Administrator
# Install the DHCP Server role
PS C:\> Install-WindowsFeature -Name DHCP -IncludeManagementTools

# Verify it installed successfully
PS C:\> Get-WindowsFeature -Name DHCP

# Complete post-install config (creates security groups)
PS C:\> Add-DhcpServerInDC -DnsName "DC01.company.local" -IPAddress "192.168.1.1"
💡 Where to install DHCP? In small environments, install DHCP on your Domain Controller. In larger networks, it's better practice to put it on a separate member server. But for labs and small offices, the DC is perfectly fine.

Creating an IP Pool (Scope)

Define the range of IPs DHCP can hand out

What is a DHCP Scope?

A Scope is a pool of IP addresses that the DHCP server manages and hands out. You define the start IP, the end IP, the subnet mask, and any exclusions (IPs to skip).

Example: "Hand out IPs from 192.168.1.100 to 192.168.1.200. Skip 192.168.1.150 because that's reserved for the printer."

Plan Your IP Range First

Before creating a scope, plan your address space. Here's a typical small office layout:

RangePurpose
192.168.1.1 – 192.168.1.10Network infrastructure (router, switches)
192.168.1.11 – 192.168.1.50Servers (static/manually assigned)
192.168.1.51 – 192.168.1.99Printers & static devices
192.168.1.100 – 192.168.1.200DHCP pool for workstations ← This is your scope
192.168.1.201 – 192.168.1.254Spare / future use

Create a Scope via DHCP Manager (GUI)

1

Open DHCP Manager

Start → Windows Administrative Tools → DHCP. Or press Win+R → dhcpmgmt.msc

2

Right-click IPv4 → New Scope

Expand your server → right-click IPv4 → New Scope…

3

Name the Scope

Give it a descriptive name like "Office Workstations" and optional description.

4

Set IP Range

Start IP: 192.168.1.100   End IP: 192.168.1.200   Mask: 255.255.255.0

5

Add Exclusions (optional)

If any IPs in that range are already used (e.g., you manually set a device to 192.168.1.150), add them as exclusions so DHCP won't hand them out.

6

Set Lease Duration

Default is 8 days. Fine for most offices. For guest WiFi or cafes, use 2-4 hours. Longer leases = less network traffic. Shorter = IPs return to pool faster.

7

Configure Scope Options

Select "Yes" to configure now → Enter:
Router (gateway): 192.168.1.1
DNS Server: 192.168.1.1 (your DC)

8

Activate the Scope

Select "Yes, I want to activate this scope now." Click Finish. The scope is now live!

Create a Scope via PowerShell

PowerShell — All in one go
# Create the scope
PS C:\> Add-DhcpServerv4Scope -Name "Office Workstations" -StartRange "192.168.1.100" -EndRange "192.168.1.200" -SubnetMask "255.255.255.0" -State Active

# Set scope options (Gateway + DNS)
PS C:\> Set-DhcpServerv4OptionValue -ScopeId "192.168.1.0" -Router "192.168.1.1" -DnsServer "192.168.1.1" -DnsDomain "company.local"

# Verify the scope exists
PS C:\> Get-DhcpServerv4Scope
💡 DHCP Reservations: Want a device to always get the same IP but still use DHCP? Create a reservation — you tie the device's MAC address to a specific IP. The device still uses DHCP, but always receives the same IP. Best of both worlds!
# Create a reservation for a printer (tie MAC to IP)
PS C:\> Add-DhcpServerv4Reservation -ScopeId "192.168.1.0" -IPAddress "192.168.1.150" -ClientId "AA-BB-CC-DD-EE-FF" -Description "Reception Printer"

Authorizing DHCP in the Domain

Security mechanism to prevent rogue DHCP servers

Why Does DHCP Need Authorization?

Imagine someone plugs a personal router into your office network. That router has its own DHCP server and starts handing out wrong IP addresses to your computers — pointing DNS to the wrong server, hijacking internet traffic, or causing IP conflicts. This is called a Rogue DHCP Server.

Microsoft's solution: in an Active Directory domain, DHCP servers must be authorized in Active Directory before they'll hand out any leases. Unauthorized DHCP servers are automatically suppressed.

How Authorization Works

①
DHCP Starts
Server checks if it's in a domain
→
②
Queries AD
"Am I on the authorized list?"
→
✓
Authorized
Starts handing out leases
or
✗
Unauthorized
Shuts down immediately

Authorize via DHCP Manager (GUI)

1

Open DHCP Manager

Win+R → dhcpmgmt.msc

2

Look for the Red Arrow (unauthorized)

An unauthorized server shows a red down-arrow icon next to its name. An authorized server shows a green up-arrow.

3

Authorize

Right-click on your server name (at the top of the tree) → Authorize. Wait a few seconds, then press F5 to refresh — the arrow should turn green.

Authorize via PowerShell

PowerShell — Run as Domain Admin
# Authorize the DHCP server in AD
PS C:\> Add-DhcpServerInDC -DnsName "DC01.company.local" -IPAddress "192.168.1.1"

# Verify which DHCP servers are authorized in the domain
PS C:\> Get-DhcpServerInDC

# Expected output:
# IPAddress DnsName
# --------- -------
# 192.168.1.1 DC01.company.local

Testing That Everything Works

On a Client PC — Command Prompt
# Release current IP and request a new one from DHCP
C:\> ipconfig /release
C:\> ipconfig /renew

# Show your IP configuration — confirm you got an IP in your scope range
C:\> ipconfig /all

# You should see:
# IPv4 Address: 192.168.1.101 (from your scope!)
# DHCP Server: 192.168.1.1 (your DC)
# DNS Servers: 192.168.1.1
💡 View Active Leases: In DHCP Manager, expand your scope → Address Leases. You'll see every device that has received an IP, their MAC address, and when the lease expires. This is incredibly useful for network troubleshooting.
